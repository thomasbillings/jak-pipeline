# Code of Conduct

`jak-pipeline` follows the [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) as its code of conduct.

## Summary

- **Be welcoming and respectful.** Different people have different experiences, abilities, and points of view. A productive collaboration assumes good faith and engages with what someone said rather than who they are.
- **Be constructive.** Critique ideas, code, and decisions — not the person behind them. Sharp technical disagreement is welcome and useful; personal attacks are not.
- **Be patient with newcomers.** Everyone was new once. A clear explanation today saves the same question being asked next week.
- **Don't harass anyone.** Demeaning, intimidating, or sustained-unwelcome behaviour toward any contributor or maintainer is not acceptable.

## Enforcement

If you experience or witness behaviour that violates this code of conduct, please report it privately to:

- **Email:** `thomas.billings@gmail.com` with subject `jak-pipeline conduct:`

Reports are reviewed and acted on as quickly as possible. Confidentiality is maintained for the reporter unless required by law or with their explicit permission.

Consequences for violations follow the Contributor Covenant v2.1 enforcement ladder (Correction → Warning → Temporary Ban → Permanent Ban). The maintainer has final discretion on the appropriate response to a given incident.

## Scope

This Code of Conduct applies within all project spaces — issues, pull requests, discussion threads, commit messages, code comments, and any direct communication about the project — and also when an individual is representing the project in public.
